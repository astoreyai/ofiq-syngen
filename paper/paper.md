# Summary

`ofiq-syngen` generates controlled synthetic degradations of face images
that target the 28 quality components defined in ISO/IEC 29794-5 [@iso29794_5_2024].
Each perturbation operates on the same image regions that the OFIQ
reference implementation [@schlett2024ofiq] measures, so a degraded
image scored by OFIQ produces predictable per-component score
movements. The package is the canonical synthetic-degradation companion
to OFIQ, intended for training quality-aware face recognition systems,
calibrating quality measurement implementations, and preparing
operationally relevant test sets for ICAO Doc 9303 Part 9 conformance
[@icao9303_part9].

# Statement of need

ISO/IEC 29794-5:2024 specifies face image quality as a vector of 28
components (illumination uniformity, sharpness, head pose, occlusion,
and so on), and the OFIQ reference implementation operationalizes the
measurement of those components. Training and validating systems
against this standard requires data with controlled, known
quality degradations. Three gaps exist in the current toolset:

1. Existing degradation libraries (GFPGAN [@gfpgan], Real-ESRGAN
   [@realesrgan], BSRGAN [@bsrgan], NatOcc [@natocc]) operate on pixels
   without targeting the specific image regions ISO/IEC 29794-5
   measures. A whole-image gamma correction moves the OFIQ
   `LuminanceMean` score, but it also moves five other component
   scores; the perturbation is not isolated to the targeted component.
2. Existing libraries lack the per-component severity control needed
   for the calibration sweeps required by OFIQ-aligned validation
   work. The systematic review by [@p6_systematic_review] found that
   zero of 61 surveyed FIQA papers fit a bounded regression of FIQA
   score against a controllable degradation parameter.
3. No public tool maps every perturbation to ISO/IEC 19794-5 or ICAO
   Doc 9303 Part 9 clauses, blocking operational deployers (passport
   issuance, eMRTD enrollment, border control) from connecting modern
   measurement to operational standards.

`ofiq-syngen` addresses these gaps by implementing 28 degraders, one
per OFIQ component, each operating on the precise image region that
OFIQ measures. The `FaceContext` abstraction runs OFIQ's own ONNX
landmark, parsing, occlusion, and pose models once per image and
shares the result across all degraders, ensuring region targeting is
identical to OFIQ's own. A multi-standard mapping ships per component:
ISO/IEC 29794-5 aspect, ISO/IEC 19794-5 [@iso19794_5_2011] clause,
ICAO 9303 Part 9 clause, alignment classification, and confidence
flag.

# Functionality

The package exposes a Python API (`DegradationPipeline.degrade_single`,
`degrade_sweep`, `degrade_all_components`, `generate_dataset`) and a
CLI (`ofiq-syngen degrade`, `sweep`, `list-components`,
`show-standards`, `generate-dataset`, `export-conformance`). Three
standards-conformance presets (`iso-29794-5`, `iso-19794-5`,
`icao-strict`) filter the component set for targeted use cases. The
`export-conformance` subcommand bundles the standards mapping, OFIQ
source provenance, and parity manifest into a ZIP that downstream
OFIQ-aligned implementations can use to validate their own
measurement chains.

A real-time capture example (`examples/realtime_capture.py`)
demonstrates webcam-based face capture with live OFIQ-aligned quality
feedback, aggregating the 28 components into five user-facing panels
(lighting, focus, pose, expression, occlusion) and triggering capture
when all panels stay green for a configurable number of consecutive
scored frames.

# Validation

The test suite contains 301 passing tests across 11 dimensions, with
216 of those parametrized per component (every of the 28 components
gets its own row in pytest output for smoke, determinism, seed
sensitivity, severity-changes-output, severity monotonicity in image
delta, output finiteness, and size invariance over small / medium /
large fixtures).

Per-component algorithm fidelity to the OFIQ reference implementation
is documented in `docs/standards/PROVENANCE.md` with line-range
citations into the OFIQ C++ source (pinned to release `1.1.0`). The
infrastructure for OFIQ-binary score-parity testing
(`tests/test_ofiq_parity.py`, `scripts/regenerate_parity_vectors.py`)
ships in this release; the score-parity manifest is regenerated by
running the package against a built OFIQ binary, which downstream
users do once per OFIQ release upgrade.

A throughput benchmark (`benchmarks/bench_throughput.py`) measures
per-component wall-clock degradation cost; representative numbers on a
contemporary CPU range from 200 to 15,000 images per second per
component, depending on whether the perturbation requires ONNX
inference for face context.
